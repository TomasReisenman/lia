¿Qué es Lia?

Lia (Lecturas para Ia) es un Asistente Educativo Personalizado construido con una arquitectura multi-agente. Está diseñado para acompañar a estudiantes en su proceso de aprendizaje de Inteligencia Artificial y Machine Learning, ofreciendo una experiencia interactiva, adaptativa y completa.

En lugar de ser un simple chatbot que responde preguntas, Lia delega cada tarea a un agente especializado. Esto permite que cada componente del sistema esté optimizado para una función específica: buscar información en Wikipedia, encontrar proyectos en GitHub, generar ejercicios prácticos, corregir respuestas, leer artículos de la web o recomendar qué estudiar según el nivel del estudiante.

¿Cómo ayuda a un estudiante?

📚 Aprender conceptos

Pide "explícame redes neuronales" y Lia busca en Wikipedia, resume en español y adapta la explicación al nivel del estudiante.

📝 Practicar con ejercicios

Genera ejercicios de opción múltiple personalizados. El estudiante responde y Lia corrige, explicando por qué cada respuesta es correcta o incorrecta.

🎯 Planificar el estudio

Recomienda temas de IA/ML según el nivel y los intereses del estudiante, usando búsqueda semántica sobre una base de 20 temas indexados.

🔗 Explorar contenido web

Comparte un link de un artículo o tutorial y Lia lo lee, resume y genera ejercicios basados en ese contenido específico.

🛡️ Guardrail — before_model_callback

Filtro de contenido implementado como callback que se ejecuta antes de cada request al LLM. Bloquea mensajes que contienen términos prohibidos.

FORBIDDEN_TERMS

Lista de términos bloqueados

FORBIDDEN_TERMS = [
    "hack", "hacking", "crack",
    "phishing", "malware",
    "ransomware", "keylogger",
    "exploit", "vulnerabilidad",
    "inyeccion sql", "sql injection",
]

block_disallowed_content()

Callback registrado en root_agent

def block_disallowed_content(
    callback_context, llm_request
):
    last_text = llm_request.contents[-1] \
        .parts[0].text.lower()
    for term in FORBIDDEN_TERMS:
        if term in last_text:
            return LlmResponse(content=
                "Lo siento, no puedo ayudar")
    return None  # permitir

📊 Métricas — OpenTelemetry + Grafana

Se utilizó un stack Docker con Grafana, Prometheus y un servidor OTEL Collector (grafana/otel-lgtm) para capturar métricas en tiempo real del sistema multi-agente. 

docker run --name local-otel-server -p 4317:4317 -p 4318:4318 -p 3000:3000 grafana/otel-lgtm:latest

Antes de levantar ADK se exporta el endpoint:

export OTEL_EXPORTER_OTLP_ENDPOINT="http://localhost:4318"

Métricas disponibles

Tipos: count, bucket, sum

gen_ai_client_operation_duration_seconds

Duración de cada operación del cliente LLM (request → response).

gen_ai_client_token_usage

Cantidad de tokens consumidos (input y output) por cada request.

gen_ai_execute_tool_duration_seconds

Tiempo de ejecución de cada tool call (MCP, FAISS, HTTP).

gen_ai_invoke_agent_duration_seconds

Duración de la invocación de cada sub-agente (latencia por agente).

Grafana en localhost:3000 — dashboards preconfigurados para visualizar estas métricas.

🧪 Testing — LLM as Judge

Evaluación automatizada usando un LLM como juez para comparar respuestas esperadas vs reales.

adk eval . tests/evalset.json --config_file_path=tests/test_config.json

evalset.json

5 casos de prueba con input y respuesta esperada

  • case_1: Guardrail — "estudiar hacking"
  • case_2: Delegación Wikipedia — "entropia cruzada"
  • case_3: Delegación Study Advisor — "sugerir temas"
  • case_4: Delegación Exercises — "ejercicios álgebra"
  • case_5: Delegación Corrector — "corregir ejercicios"

test_config.json

Configuración del juez LLM

{
  "criteria": {
    "final_response_match_v2": {
      "threshold": 0.8,
      "judge_model_options": {
        "judge_model": "gemini-3.1-flash-lite",
        "num_samples": 2
      }
    }
  }
}
